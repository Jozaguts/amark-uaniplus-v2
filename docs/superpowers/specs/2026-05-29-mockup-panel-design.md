# Mockup Panel — Design Spec

**Date:** 2026-05-29  
**Status:** Approved

---

## Overview

Add a **Mockup** tool to the design editor's left sidebar. When active, the left panel renders a composited preview of the product with the user's design applied on top of a lifestyle mockup image (e.g., a model wearing the shirt). Includes a download button. The panel mirrors the active view of the Konva canvas in real time.

---

## API Contract

The editor response (`/editor/:id`) gains a top-level `mockups` array inside `editor`, at the same level as `views`:

```json
{
  "editor": {
    "views": [...],
    "mockups": [
      {
        "viewId": "front",
        "src": "https://cdn.example.com/mockups/front-white.png",
        "width": 3000,
        "height": 3000,
        "previewColorId": "color-white",
        "previewColorHex": "#ffffff"
      }
    ]
  }
}
```

**Rules:**
- One entry per `viewId` + `previewColorId` combination.
- Default `src` mirrors `editor.views[].mockup.src` (the flat product image). Editable by admin per view/color.
- `editor.views[].mockup` (singular) is never editable — generated by the system, used exclusively by the Konva canvas.
- `editor.mockups` is editable by admin and used exclusively by the Mockup panel.

**Props:**

| Prop | Type | Description |
|---|---|---|

## Mockup Selection Logic

In `[id].vue`, a new computed resolves the active lifestyle mockup:

```typescript
const activeMockups = computed(() => editor.value?.mockups ?? [])

const activeLifestyleMockup = computed(() => {
  const viewId = activeViewId.value
  const colorId = selectedColorId.value
  const candidates = activeMockups.value.filter(m => m.viewId === viewId)

  return candidates.find(m => m.previewColorId === colorId)
    ?? candidates[0]
    ?? null
})
```

## Compositing Logic

Runs inside `MockupPanel.vue` on a debounced watcher (120ms) triggered by changes to `activeMockup`, `designObjects`, or `printArea`.

### With lifestyle mockup (`activeMockup` is not null)

```
1. Create <canvas> sized to activeMockup.width × activeMockup.height
2. Draw activeMockup.src as background (crossOrigin: 'anonymous')
3. Compute scale factors:
     scaleX = activeMockup.width  / flatMockup.width
     scaleY = activeMockup.height / flatMockup.height
4. For each designObject in active view:
     - Normalize: objX = object.x - printArea.x
                  objY = object.y - printArea.y
     - Scale to mockup space:
         x = (printArea.x * scaleX) + (objX * scaleX)
         y = (printArea.y * scaleY) + (objY * scaleY)
         w = object.width  * scaleX
         h = object.height * scaleY
     - Apply rotation via context.rotate()
     - Draw image or text
5. Export: canvas.toDataURL('image/png') → compositeDataUrl
```

### Fallback (no `activeMockup` for active view/color)

Use `fallbackDataUrl` (the Konva stage export passed as prop). No canvas compositing needed.

## Download

```typescript
const downloadMockup = () => {
  const url = compositeDataUrl.value ?? fallbackDataUrl.value
  if (!url) return

  const a = document.createElement('a')
  a.href = url
  a.download = `mockup-${activeViewId}.png`
  a.click()
}
```

Triggers a browser download. No external APIs or server calls.

---

## Reactivity

| Trigger | Action |
|---|---|
| `activeViewId` changes | Re-select lifestyle mockup, re-composite |
| `selectedColorId` changes | Re-select lifestyle mockup (color match), re-composite |
| `designObjects` changes | Re-composite with debounce 120ms |
| User switches to Mockup tool | Render immediately if `compositeDataUrl` is stale |

The panel only renders when `activeTool === 'mockup'` to avoid unnecessary work while the user is designing.

---

## What is NOT included

- The "Adjusted mockup" switch — explicitly excluded.
- Multiple lifestyle images per view (only `mockups[0]` matching color is used).
- Uploading lifestyle images from the frontend — admin-only via backend panel.
